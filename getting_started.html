01

Quick Start for a Delivered Folder

Extract or copy the ARGUS folder to a local path such as C:\Users\you\Desktop\ads_classification. Open PowerShell inside that folder. You should see pyproject.toml, config.example.yaml, start.bat, frontend\, and ad_classifier\.

Assumption for this guide: the delivered folder does not include a working .venv. You will create a fresh environment and install the correct torch build for your GPU before installing the embedding packages.
  1. Install or verify Python, Node.js, ffmpeg, and an OpenAI-compatible inference engine.
  2. Create and activate .venv.
  3. Install ARGUS core dependencies.
  4. Install the correct torch build for NVIDIA, AMD, or CPU.
  5. Install embedding libraries without asking pip to replace torch.
  6. Copy config.example.yaml to config.yaml and set endpoints.
  7. Install OCR and frontend dependencies.
  8. Initialize SQLite and run start.bat.
02

What Must Be Installed

Python 3.11 or newer

Runs the FastAPI backend, worker, pipeline, CLI, tests, OCR, and vector code.

Node.js 18 or newer

Runs the Vite React frontend at http://127.0.0.1:5173.

ffmpeg and ffprobe

Required for video probing, frame sampling, and audio extraction.

OpenAI-compatible inference engine

Provides /v1/chat/completions for the VLM, agent, and optional GLM-OCR.

Git is optional for delivered folders

Only needed if you will pull updates from a remote repository. It is not required to run a copied project folder.

Python build tools

pip, setuptools, and wheel are upgraded inside the venv before package install.

python --version
node --version
ffmpeg -version
ffprobe -version
03

Python Environment

Create a clean local Python environment from the ARGUS project root. This keeps package installs contained inside the project folder.

If activation is blocked in the current PowerShell session:

Set-ExecutionPolicy -Scope Process RemoteSigned

Create and activate the environment:

python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip setuptools wheel
python -m pip install -e ".[dev,clustering]"

04

Torch for NVIDIA, AMD, or CPU

Install torch before the embedding libraries. Use the current official command for your GPU and Python version; wheel URLs change over time, so treat the examples below as the shape of the command and verify them against the official pages.

NVIDIA CUDA

Use the PyTorch selector at pytorch.org/get-started/locally. Choose Windows, Pip, Python, and the CUDA runtime that matches the selector recommendation for your driver. 

# Example shape only: use the exact CUDA index URL from the PyTorch selector.
python -m pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu126

AMD Radeon / ROCm on Windows

Use AMD's official Radeon and Ryzen ROCm documentation: PyTorch via PIP installation. AMD Windows wheels are tied to driver, Python, GPU generation, and ROCm release. Follow the AMD page exactly, including any required Python version and runtime packages. 

# Example shape only: copy the current wheel command from AMD's ROCm page.
python -m pip install --no-cache-dir https://repo.radeon.com/rocm/windows/.../torch-...-win_amd64.whl

CPU fallback

CPU torch works for tests and small demos but visual search and embedding backfills will be slow. Use this only when GPU setup is unavailable. 

python -m pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu

Verify torch before continuing

@'
import torch

print("torch:", torch.__version__)
print("cuda available:", torch.cuda.is_available())
print("device:", torch.cuda.get_device_name(0) if torch.cuda.is_available() else "cpu")
'@ | python -

Install embedding libraries after torch

Use --no-deps here so these libraries do not replace the torch wheel you just installed. 

python -m pip install --no-deps sentence-transformers==3.0.1 transformers==4.57.6 tokenizers==0.22.1
05

Inference Engine Configuration

ARGUS is engine-agnostic. Use any local or hosted inference engine that exposes the OpenAI-compatible chat completions API. The VLM must be vision-capable for classification and marketing entity extraction.

  1. Start the inference engine.
  2. Load or select a vision-capable model.
  3. Confirm the endpoint accepts POST /v1/chat/completions.
  4. Copy config.example.yaml to config.yaml.
  5. Set vlm.local.endpoint and vlm.local.model.

The copied config is intentionally commented with the selectable values and tuning notes. Use config.yaml as the setup reference after copying it. 

Copy-Item .\config.example.yaml .\config.yaml
vlm:
  mode: local
  local:
    endpoint: "http://127.0.0.1:1234/v1"
    model: "your-vision-model"
    response_format: json_object

When agent.inherit_vlm: true, the campaign research agent and natural-language agent use the same active OpenAI-compatible endpoint as the classifier.

JSON response format

FormatUse whenTradeoff
json_object Most local or quantized engines. Requires prompt discipline but is broadly compatible.
json_schema The engine supports OpenAI Structured Outputs. Stronger validation, but unsupported engines may fail or ignore it.
06

OCR, GLM-OCR, and Whisper

PaddleOCR

PaddleOCR is the grounded local OCR engine. It stores raw text, boxes, confidence, engine, frame, and timestamp.

python -m pip install paddlepaddle
python -m pip install paddleocr
python -m pip install -e ".[ocr,dev]"

Optional GLM-OCR

GLM-OCR is optional and is called only for gated hard frames: dense text, low confidence OCR, tiny fragments, or difficult end cards. It should be served by an OpenAI-compatible endpoint. 

glm_ocr:
  enabled: true
  mode: local
  include_in_search: true
  include_in_vlm_bundle: false
  local:
    endpoint: "http://127.0.0.1:5050/v1"
    model: "glm-ocr"

Whisper

The repository includes tools\whisper.cpp\whisper-cli.exe. The model file is not included. Place a GGML model under models\whisper\ and point config.yaml at it. 

New-Item -ItemType Directory -Force .\models\whisper
07

Initialize and Run

Install frontend dependencies once:

cd frontend
npm install
cd ..

Initialize SQLite, then start the API, worker, and frontend:

python -m ad_classifier init-db
.\start.bat
API http://127.0.0.1:8000
API docs http://127.0.0.1:8000/docs
Frontend http://127.0.0.1:5173

Remote viewer testing

If you expose ARGUS through a temporary tunnel for another person to test, expose only the frontend on port 5173. The browser will keep API and media requests on that same origin, and Vite forwards /api and /data to the local FastAPI service at http://127.0.0.1:8000.

Leave VITE_API_BASE_URL blank for this mode. Only set VITE_API_PROXY_TARGET if the API is running on a different local host or port.

Basic Auth for a demo domain

Basic Auth is disabled by default. For a temporary public demo domain, enable it in frontend\.env.local before starting Vite. The file is ignored by git. 

cd frontend
Set-Content .env.local @"
ARGUS_BASIC_AUTH_ENABLED=true
ARGUS_BASIC_AUTH_USERS=h-tech:change-this-password,analyst:another-password
ARGUS_BASIC_AUTH_REALM=ARGUS Demo
"@
npm run dev

Add users by adding comma-separated username:password pairs, then restart Vite. Use HTTPS on the public domain and keep the FastAPI port private.

Troubleshooting

The app starts but upload fails

Confirm the inference engine is running and the configured VLM model supports images.

Visual search is empty

Install the embedding packages without changing torch, then run python -m ad_classifier reindex-visual-frames for old ads.

PaddleOCR import fails

Install paddlepaddle first, then paddleocr, then reinstall the OCR extra.

Slow VLM responses

Use streaming on the inference endpoint and raise endpoint timeouts for remote engines with long time-to-first-token.
