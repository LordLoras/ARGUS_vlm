# Intelligence Crawler Normalized Ad Schema

This is the API-facing ad-resource shape for digital ad library captures. It is
computed at read time from `intel_resources` columns plus the raw adapter payload
stored in `metadata_json`.

The design goal is to keep provider facts without making Google, Meta, YouTube,
or future adapters leak their private field names into downstream analysis.

## Contract

Each resource returned by `GET /api/intelligence/resources` includes:

- `metadata`: raw adapter/provider data. This is for debugging, replay, and fields
  we have not normalized yet. It should not be the primary contract for analysis.
- `normalized`: stable cross-provider data. This is what API clients, campaign
  analysis, and later TV-ad matching should prefer.

## Top-level resource fields

| Field | Meaning |
|---|---|
| `id` | Local crawler resource id. Stable inside `intelligence_crawler.db`. |
| `brand_name` | Watched brand attached to the source. This may differ from provider advertiser text. |
| `source_id` | Local source config id that produced this resource. |
| `source_type` | Adapter name, for example `google_atc` or `meta_ad_library_ui`. |
| `resource_type` | Local resource category, for example `atc_ad` or `meta_ad`. |
| `url` | Provider/library URL for the creative when known. |
| `platform` | Source platform projection, for example `meta`; nullable. |
| `platform_id` | Provider creative/library id when the adapter has one. |
| `title` | Human-readable local title generated by the adapter. |
| `description` | Best available text/copy summary from the adapter. |
| `published_at` | Provider first-seen/started date when known. |
| `first_seen_at` | Time our crawler first observed the resource. |
| `fetched_at` | Time our crawler last fetched/refreshed the resource. |
| `is_backfill` | True when the item was recorded during baseline/backfill and did not emit a live signal. |
| `variant_count` | Provider-reported creative variant count when known. |
| `has_variants` | True when the provider says the creative has multiple versions or the count is greater than one. |
| `artifact_summary` | Counts for screenshots, image URLs, video URLs, posters, background images, links, and stored media assets. |
| `artifacts` | Flat artifact list for UI display and downloads/previews. |
| `metadata` | Raw adapter payload, preserved as collected. |
| `normalized` | Provider-agnostic creative object described below. |

## `normalized`

| Field | Meaning |
|---|---|
| `provider` | Human provider family. Current values are `google_ads_transparency`, `meta_ad_library`, or the adapter source if unknown. |
| `adapter` | Exact local adapter that produced the row. |
| `platform` | Platform projection from the local source, when configured. |
| `source_url` | URL that led to the capture, such as a Meta library URL or Google ATC start URL. |
| `fetched_at` | Same fetch timestamp as the resource row, copied here for standalone API consumers. |
| `advertiser` | Normalized advertiser identity. |
| `creative` | Normalized creative identity, dates, format, and variant flags. |
| `variants` | One or more normalized creative versions. If the provider has no explicit variants, a `Default creative` variant is generated from the row text and artifacts. |
| `delivery` | Provider delivery stats such as regions, surfaces, and impression ranges. Empty when the provider does not expose this data. |
| `targeting` | Provider targeting disclosures. `raw` preserves the provider category object; `decoded` is reserved for a later human-readable decoder. |
| `collection` | Context about how we queried or collected the resource. This is separate from delivery stats. |

## `normalized.advertiser`

| Field | Meaning |
|---|---|
| `id` | Provider advertiser id, for example a Google `AR...` id. |
| `name` | Provider advertiser name when present; otherwise the watched brand name. |
| `url` | Provider advertiser/library page. |

## `normalized.creative`

| Field | Meaning |
|---|---|
| `id` | Provider creative/library id, for example Google `CR...` or Meta library id. |
| `library_url` | Direct provider URL for this creative when known. |
| `format` | Lowercase creative format such as `image`, `video`, or `rich_media`. |
| `status` | Provider status when exposed, for example Meta active/inactive text. Null otherwise. |
| `title` | Local adapter title. |
| `description` | Best available creative copy summary. |
| `first_shown_at` | Provider first-shown/started-running timestamp when known. |
| `last_shown_at` | Provider last-shown timestamp when known. |
| `served_days` | Provider-supplied number of served days. We do not compute this from first/last dates because providers may count only active delivery days. |
| `variant_count` | Provider-reported number of variants when known. |
| `has_variants` | True when variants are known or inferred from the provider count. |

## `normalized.variants[]`

| Field | Meaning |
|---|---|
| `id` | Provider variant id if present. |
| `label` | Human label such as `Variant 1` or `Default creative`. |
| `description` | Variant body/copy/headline text. |
| `cta` | Call to action text, for example `INSTALL`, `Shop now`, or link text. |
| `landing_url` | Click-through destination when exposed. |
| `assets` | Images, videos, screenshots, posters, background images, and clickout links attached to this variant. |

## `normalized.variants[].assets[]`

| Field | Meaning |
|---|---|
| `asset_type` | Broad type: `image`, `video`, `landing_page`, or adapter-specific fallback. |
| `role` | How the asset is used: `creative`, `screenshot`, `video_poster`, `background`, `clickout`, or `supporting`. |
| `url` | Remote URL for provider-hosted media or landing pages. |
| `path` | Local path for crawler-created artifacts such as screenshots. |
| `text` | Link text or artifact label when present. |
| `source` | `provider` for provider URLs, `crawler` for locally captured files. |

## `normalized.delivery`

| Field | Meaning |
|---|---|
| `regions` | Provider delivery regions. Empty when unknown. A single region with only `region_code` may be present when the adapter only knows the requested market. |

## `normalized.delivery.regions[]`

| Field | Meaning |
|---|---|
| `region_code` | Delivery region code such as `US` or `DE`. |
| `region_name` | Human region name such as `Germany`, when supplied. |
| `first_shown_at` | First-shown timestamp for that region when supplied. |
| `last_shown_at` | Last-shown timestamp for that region when supplied. |
| `impressions` | Provider impression range for the region. |
| `surfaces` | Delivery surfaces inside the region, for example YouTube or Search. |

## `normalized.delivery.regions[].surfaces[]`

| Field | Meaning |
|---|---|
| `surface_code` | Provider surface code such as `YOUTUBE` or `SEARCH`. |
| `surface_name` | Human surface name. |
| `impressions` | Provider impression range for that surface. |

## `normalized.delivery.*.impressions`

| Field | Meaning |
|---|---|
| `lower_bound` | Lower provider estimate for impressions. |
| `upper_bound` | Upper provider estimate for impressions. |

## `normalized.targeting`

| Field | Meaning |
|---|---|
| `raw` | Provider targeting disclosure object as collected. |
| `decoded` | Reserved for a later decoder that converts provider category codes into plain labels. |

## `normalized.collection`

| Field | Meaning |
|---|---|
| `requested_region_code` | Region requested in the provider query, for example `US` or `DE`. |
| `collector_region_code` | Region where the collector VM/browser is running, if we later run region-specific collectors. |
| `collector_id` | Stable id for a collector process/VM, if configured later. |
| `source_url` | Query/library URL used for collection. |

## Region fields: implement now or later?

Implement them now as optional/empty structures. They are already in the normalized
schema because region data has two separate meanings:

1. Delivery region: where the provider says the ad served.
2. Collection context: where and how our crawler asked for the data.

Keeping both concepts now prevents a breaking API change later if we run US, EU,
or other region VMs. For current single-region crawls, use:

```json
{
  "delivery": { "regions": [{ "region_code": "US" }] },
  "collection": {
    "requested_region_code": "US",
    "collector_region_code": null,
    "collector_id": null
  }
}
```

When region-specific collectors exist, populate `collector_region_code` and
`collector_id`; do not overload `delivery.regions` with VM location.
