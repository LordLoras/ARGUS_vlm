01

System Mission

ARGUS converts local video advertisements into structured, searchable, analyst-ready records. It is categorization-only: the system does not approve, block, gate, or escalate ads. Observation tags such as urgency, hidden disclaimers, or before/after claims are descriptive search fields, not policy decisions.

Local first

SQLite, local artifacts, local worker, local-first model endpoints. No Docker is required by default.

Evidence grounded

OCR, transcripts, rules, frames, VLM evidence, and marketing entities retain source context.

Analyst usable

Search, campaign grouping, research briefs, and the agent expose practical marketing analysis value.

Architecture Overview

The backend owns ingestion, persistence, retrieval, campaign logic, and agent tools. The frontend is decoupled and communicates through JSON endpoints and SSE streams.

InputVideo upload and metadata
WorkerFrames, audio, OCR, transcript
ReasoningRules, VLM verification, entities
IndexFTS5, text vectors, frame vectors
SurfaceAPI, React UI, read-only agent
LayerImplementationNotes
APIFastAPIJSON endpoints and SSE streams for job and agent progress.
WorkerPython stage orchestrationRestartable, logs every degraded path, reuses artifacts unless forced.
DatabaseSQLite WALConcurrent API reads and worker writes, with FTS5 and sqlite-vec.
FrontendVite, React, TypeScriptConsumes HTTP/SSE only; dev and preview proxy /api and /data to the local API.
InferenceOpenAI-compatible engineVLM, agent, OCR cleanup, self-correction, optional GLM-OCR.

For temporary remote testing, the frontend is the only service that needs to be exposed. Keeping browser requests same-origin avoids rotating API tunnel URLs and avoids widening FastAPI CORS for short-lived hosts. VITE_API_BASE_URL is reserved for deliberate split-origin deployments; otherwise the Vite proxy sends /api and /data to http://127.0.0.1:8000. Custom domains should be allowed with VITE_ALLOWED_HOSTS in frontend\.env.local. A leading dot such as .argus.rest allows the apex domain and subdomains.

The Vite edge can also enforce optional demo auth before serving the UI, proxied API, or media files. It is controlled by ARGUS_AUTH_ENABLED, ARGUS_AUTH_MODE, and ARGUS_AUTH_USERS in frontend\.env.local. Users are comma-separated username:password pairs. ARGUS_AUTH_MODE=login shows the ARGUS-themed login page; basic uses the browser-native Basic Auth prompt. This is meant for temporary HTTPS demos while the FastAPI port stays private.

03

Pipeline Stages

STAGE 01

Upload and Job Creation

POST /api/ads/upload writes the video to data/uploads/, computes SHA256 source_hash, creates an ads row, and queues a jobs row. Progress is streamed over /api/jobs/{id}/events.

STAGE 02

Ingest Artifacts

ffmpeg/ffprobe probe duration, dimensions, frame rate, sample frames at the configured interval, and extract mono 16 kHz audio. Whisper produces transcript segments. The manifest is sorted by explicit frame index and timestamp, never filesystem order.

STAGE 03

Frame Preprocessing

Frames are checked for blankness, blur, contrast, perceptual hash duplication, and scene changes. Kept frames feed OCR, visual embeddings, VLM bundles, and visual search.

STAGE 04

OCR and Conditional Document OCR

PaddleOCR is the raw grounded OCR engine. Optional GLM-OCR is called only for gated hard frames, such as dense text, low-confidence OCR, fragmented text, blurry frames, or fine print. Raw PaddleOCR and GLM-OCR text are stored separately in ocr_items.

STAGE 05

Transcript Alignment and Rules

Whisper segments are aligned to frames with a configurable time window. Deterministic rules over OCR and transcript text produce category evidence and descriptive observation tags. Rules never gate content.

STAGE 06

Evidence Bundle

The worker builds a compact chronological bundle from selected frames, OCR, GLM-OCR corrections, transcript excerpts, metadata, and rule triggers. Frame selection is deterministic and favors first/last frames, rule-trigger frames, dense OCR frames, and time-distributed coverage.

STAGE 07

VLM Verification and Entity Extraction

A vision-capable OpenAI-compatible model verifies evidence, classifies against the configured taxonomy, extracts marketing entities, evaluates OCR quality, cites timestamps, and returns strict JSON. Complexity detection raises token budgets for dense fine-print ads.

STAGE 08

Post-Processing and Aggregation

Parsed VLM output is validated against observed evidence. Optional OCR cleanup, self-correction, and visual verification refine the output. Final aggregation merges VLM and rule evidence into the persisted classification.

STAGE 09

Embeddings and Indexing

MiniLM creates one text vector per ad. SigLIP 2 creates one vector per kept keyframe plus a mean-pooled ad-level visual vector. SQLite FTS5 and sqlite-vec support keyword, semantic, visual, and hybrid search.

STAGE 10

Campaigns, Similarity, and Agent Surfaces

Similar ads are reported without merging. Campaign discovery proposes clusters from repeated campaign signals and brand scoped visual vectors. The read-only agent answers questions through a fixed tool catalog and audited sessions.

05

Persistence and Data Shape

SQLite is the system of record. The JSON columns remain authoritative, while projection columns on ads make list views and filters fast.

ads

Source path, status, duration, dimensions, hashes, brand/product/category projections, duplicate metadata.

frames

Frame path, timestamp, pHash, kept/dropped markers, blur/blank indicators.

ocr_items

Engine-specific OCR text, confidence, bounding boxes, frame source.

transcript_segments

Timestamped transcript evidence from Whisper backends.

classifications

Primary category, confidence, observation tags, OCR quality, evidence JSON.

marketing_entities

Brand, advertiser, products, offers, prices, CTAs, social proof, disclaimers, contact data.

campaigns / ad_campaigns

User and auto campaign records, assignment source, similarity scores.

agent_sessions / agent_messages

Audited agent runs, messages, tool calls, and tool results.

Vector tables

vec_ads_text, vec_ads_visual, and vec_frames_visual store sqlite-vec embeddings. A helper table maps frame vector keys back to ad IDs, frame indexes, timestamps, and paths.

Search and Ranking

ModeMain signalBest for
KeywordFTS5Brands, offers, phone numbers, exact terms, OCR text.
Text vectorMiniLM cosineSemantic text similarity across ads.
VisualSigLIP text-to-imageObjects, scenes, people, layouts, frame-level matches.
HybridFTS + MiniLM + RRFGeneral analyst search where terms and meaning both matter.
Visual hybridKeyword grounding + SigLIPSpecific visual queries that also have OCR/entity evidence.

Visual query behavior

Typed visual queries expand into a small phrase set. ARGUS embeds each phrase separately, searches frame vectors for each phrase, then keeps the best cosine score per ad. This avoids diluting car by averaging it with variants such as automobile or vehicle. Broad category terms also use query-intent filtering to reduce unrelated results after the visual score gate.

Rank fusion

Hybrid search uses reciprocal rank fusion over keyword and vector ranks. Keyword-first behavior is used when exact FTS hits are strong enough for a query, preventing unrelated vector-only noise from pushing out grounded results.

Duplicate and Related-Ad Logic

LayerSignalPurpose
Exact file hashSHA256 bytesCatches identical uploads. Re-encoding changes the hash.
Mean pHashPerceptual frame hash summaryCatches near-identical creative, but may miss small end-card changes.
Post-OCR duplicateFrame pHashes, OCR, transcript, commercial signatureCatches re-encoded copies with same text and offer signature.
Semantic related adsText and visual vectorsFinds same campaign, variant, or related creative without merging rows.

Post-OCR duplicate detection is intentionally conservative. Two ads with almost identical footage but different products, prices, or end cards should remain separate ads and become campaign variants, not exact duplicates.

08

Campaign Discovery and Research

Campaign discovery is user-triggered. It scans repeated campaign signals from marketing entities first, then brand-scoped visual vector clusters. No campaign names are hardcoded. Accepted proposals become user-owned assignments and are protected from later automatic rediscovery.

Discovery signals

Repeated VLM-extracted campaign names, shared offers, brand-scoped visual cohesion, assignment confidence.

Manual controls

Create, edit, delete, assign ads, remove ads, and accept reviewed proposals as curated campaign truth.

Research agent

Builds a local evidence bundle and asks the configured LLM for findings, creative review, edits, open questions, and answers.

Future expansion

The request contract already carries web and thinking flags for later advertiser-site or competitor research.

09

Read-Only Agent Design

The agent is tool-calling, not raw text-to-SQL by default. It has a fixed tool catalog for list, count, aggregate, campaign detail, ad detail, FTS search, vector similarity, hybrid search, and bounded read-only SQL.

  • Database connections use PRAGMA query_only = ON.
  • SQL escape hatch is capped and audited.
  • Agent sessions and tool calls are stored in agent_sessions and agent_messages.
  • Answers cite real ad_id and campaign_id values only.
Research answers are local first. Campaign research currently uses local ads, OCR, GLM-OCR, transcripts, classifications, entities, and assignment scores. Internet access is intentionally out of scope until a controlled web research layer is added.

json_object vs json_schema

ARGUS asks the VLM for structured JSON. The selected response_format determines how much enforcement the inference endpoint provides.

ModeWhat it meansBest fitFailure mode
json_objectOlder JSON mode. The endpoint is asked to emit a valid JSON object.Local, quantized, or OpenAI-compatible engines that do not implement schema validation.Output can still be structurally wrong, so ARGUS validates and may self-correct or fall back.
json_schemaStructured Outputs. A schema is sent and capable endpoints validate against it.Engines that explicitly support OpenAI-style schema response formats.Unsupported endpoints may return errors, empty content, or ignore the schema.
 
vlm:
  local:
    response_format: json_object
  remote:
    response_format: json_schema

Fresh environment dependency order

Torch is installed separately because NVIDIA, AMD, and CPU wheels differ. The safe order is: ARGUS core dependencies, hardware-specific torch, embedding libraries with --no-deps, OCR packages, frontend packages. 

python -m pip install -e ".[dev,clustering]"
# Install torch for NVIDIA, AMD, or CPU using the hardware-specific official command.
python -m pip install --no-deps sentence-transformers==3.0.1 transformers==4.57.6 tokenizers==0.22.1
python -m pip install paddlepaddle paddleocr
cd frontend
npm install

Final Output Contract

Final classification output is strict JSON and includes category, confidence, observation tags, evidence, OCR quality, marketing entities, campaigns, embedding metadata, related ads, raw model outputs, and debug records. 

{
  "ad_id": "ad_xxxxxxxx",
  "primary_category": "automotive",
  "risk_labels": ["rate_disclosure"],
  "confidence": 0.92,
  "evidence": [
    {
      "time_ms": 14000,
      "frame_index": 28,
      "source": "ocr",
      "text": "$299/mo for 36 months",
      "confidence": 0.87
    }
  ],
  "marketing_entities": {
    "brand": {"name": "Jeep", "logo_present": true},
    "products": ["Grand Cherokee", "Wrangler"],
    "offers": ["$299/mo 36-mo lease"],
    "ctas": ["Shop now"],
    "disclaimers": []
  },
  "related_ads": {
    "exact_duplicate_of": null,
    "near_duplicate_of": null,
    "semantically_similar": []
  }
}

Projection columns on ads are written in the same transaction as the JSON source of truth. The projections exist for fast listing and filtering; the JSON remains authoritative.
